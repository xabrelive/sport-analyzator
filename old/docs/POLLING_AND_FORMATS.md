# Форматы матчей и частота опроса

## Форматы матчей (настольный теннис)

В модели матча задаётся:

| Параметр          | Описание                    | Примеры                          |
|-------------------|-----------------------------|----------------------------------|
| `sets_to_win`     | Сетов для победы в матче    | 2 (BO3), 3 (BO5), 4 (BO7)        |
| `points_per_set`  | Очков до победы в сете      | 11 (классика), 6 (короткие сеты) |
| `win_by`          | Минимальный отрыв в сете    | 2                                |
| `is_doubles`      | Парная игра                 | true / false                     |

Типичные комбинации:

- **3 сета (BO3), до 11** — стандарт многих турниров: `sets_to_win=2`, `points_per_set=11`
- **5 сетов (BO5), до 11** — финалы и т.п.: `sets_to_win=3`, `points_per_set=11`
- **5 сетов до 6 очков** — укороченный формат: `sets_to_win=3`, `points_per_set=6`
- **7 сетов** — длинный матч: `sets_to_win=4`, `points_per_set=11`
- **Парные** — та же логика по сетам/очкам, `is_doubles=true` (home/away — пары).

Формат хранится в `matches.sets_to_win`, `points_per_set`, `win_by`, `is_doubles` и используется в Probability Engine для расчёта P(win set) и P(win match).

## Частота опроса для live

Настольный теннис — быстрая игра: одно очко в среднем каждые **5–15 секунд**. Чтобы показатели (счёт, коэффициенты, вероятность) были в реальном времени:

| Режим     | Рекомендуемый интервал | По умолчанию в проекте |
|-----------|-------------------------|-------------------------|
| **Live**  | 5–15 сек                | **10 сек**              |
| Pre-match | 1–2 мин                 | 60 сек                  |

Почему 10 сек для live:

- Чаще (5 сек) — выше нагрузка на API провайдера и риск лимитов.
- Реже (15–20 сек) — можно пропустить очко или сдвиг коэффициента.
- 10 сек — разумный компромисс: задержка в пределах одного розыгрыша.

Настройка в `.env`:

```env
LIVE_POLL_INTERVAL_SECONDS=10
PREMATCH_POLL_INTERVAL_SECONDS=60
```

Celery Beat запускает сбор коэффициентов по расписанию `LIVE_POLL_INTERVAL_SECONDS`. Если провайдер отдаёт live по WebSocket, можно перейти на подписку и не опрашивать по таймеру для live.

## Итог

- В коде учтены форматы: **3/5/7 сетов**, **до 11 или до 6 очков**, **парные**.
- Для ощущения «в реал-тайме» при опросе по HTTP достаточно **~10 сек** для live и **~60 сек** для pre-match.
